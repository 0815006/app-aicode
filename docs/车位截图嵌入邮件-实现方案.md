## 二、新建接口：`ParkingScreenshotService`

```java
package com.bocfintech.allstar.service;

import com.microsoft.playwright.Browser;

/**
 * 停车大屏截图服务
 * <p>
 * 根据车位位置文本（如"4号楼B1层B124"），打开本地 ParkingScreen 页面，
 * 截取车位图 SVG 区域，写入系统剪贴板，供邮件粘贴使用。
 * </p>
 */
public interface ParkingScreenshotService {

    /**
     * 截图车位图的 SVG 区域并返回截图文件路径
     *
     * @param browser         Playwright Browser 实例（已连接 CDP）
     * @param parkingPosition 车位位置文本，如 "4号楼B1层B124"、"5号楼B2层B区B045"
     * @return 截图 PNG 文件路径，失败返回 null
     */
    String takeScreenshot(Browser browser, String parkingPosition);

    /**
     * 将图片文件写入 Windows 系统剪贴板
     *
     * @param imagePath PNG 图片的绝对路径
     * @return true 成功，false 失败
     */
    boolean copyToClipboard(String imagePath);
}
```

### 设计要点

| 要点 | 说明 |
|------|------|
| `takeScreenshot` 返回 `String` 而非 `void` | 调用方拿到路径后可决定是否清理文件 |
| `copyToClipboard` 独立方法 | 截图和写剪贴板分两步，方便调试（截了图先看看对不对，再粘贴） |
| `Browser` 参数 | 传入已连接的 Browser，不自己管理 Chrome 生命周期 |

### 3.2 `takeScreenshot()` 实现

```java
@Override
public String takeScreenshot(Browser browser, String parkingPosition) {
    if (parkingPosition == null || parkingPosition.isEmpty()) {
        log.warn("截图服务：parkingPosition 为空，跳过");
        return null;
    }

    // ---- 1. 解析 floor 和 space ----
    String floorKey = null;
    String floorId = null;
    for (java.util.Map.Entry<String, String> entry : FLOOR_MAP.entrySet()) {
        if (parkingPosition.contains(entry.getKey())) {
            floorKey = entry.getKey();
            floorId = entry.getValue();
            break;
        }
    }
    if (floorId == null) {
        log.warn("截图服务：无法识别楼层，parkingPosition={}", parkingPosition);
        return null;
    }
    String spaceLabel = parseSpaceLabel(parkingPosition, floorKey);

    // ---- 2. 拼接 URL ----
    String url = PARKING_SCREEN_URL + "?floor=" + floorId + "&space=" + spaceLabel;
    log.info("截图服务：打开页面 {}", url);

    // ---- 3. 新建独立 BrowserContext（不影响邮箱页） ----
    BrowserContext screenshotCtx = null;
    try {
        screenshotCtx = browser.newContext(
            new Browser.NewContextOptions().setViewportSize(1920, 1080));
        Page screenshotPage = screenshotCtx.newPage();

        // ---- 4. 导航并等待渲染 ----
        screenshotPage.navigate(url);
        screenshotPage.waitForLoadState(LoadState.NETWORKIDLE);
        screenshotPage.waitForTimeout(SCREENSHOT_WAIT_MS);

        // ---- 5. 截取 SVG 区域 ----
        String screenshotPath = SCREENSHOT_NAME_PREFIX
                + System.currentTimeMillis() + ".png";
        screenshotPage.locator(SVG_WRAPPER_SELECTOR).screenshot(
            new Locator.ScreenshotOptions().setPath(Paths.get(screenshotPath)));
        log.info("截图服务：截图成功 → {}", screenshotPath);
        return screenshotPath;

    } catch (Exception e) {
        log.error("截图服务：截图失败 parkingPosition={}", parkingPosition, e);
        return null;
    } finally {
        if (screenshotCtx != null) {
            try { screenshotCtx.close(); } catch (Exception ignored) {}
        }
    }
}
```

### 3.3 `copyToClipboard()` 实现

```java
@Override
public boolean copyToClipboard(String imagePath) {
    if (imagePath == null) return false;
    File file = new File(imagePath);
    if (!file.exists()) {
        log.warn("截图服务：剪贴板写入失败，文件不存在 {}", imagePath);
        return false;
    }
    try {
        BufferedImage image = ImageIO.read(file);
        if (image == null) {
            log.warn("截图服务：无法读取图片 {}", imagePath);
            return false;
        }
        // 匿名 Transferable：包装 BufferedImage
        Transferable transferable = new Transferable() {
            @Override public DataFlavor[] getTransferDataFlavors() {
                return new DataFlavor[] { DataFlavor.imageFlavor };
            }
            @Override public boolean isDataFlavorSupported(DataFlavor flavor) {
                return DataFlavor.imageFlavor.equals(flavor);
            }
            @Override public Object getTransferData(DataFlavor flavor) {
                return image;
            }
        };
        Toolkit.getDefaultToolkit()
               .getSystemClipboard()
               .setContents(transferable, null);
        log.info("截图服务：图片已写入系统剪贴板");
        return true;
    } catch (Exception e) {
        log.error("截图服务：剪贴板写入失败", e);
        return false;
    }
}
```

### 3.4 解析楼层和车位编号

与前端 [`goToParkingScreen()`](web-aicode-vue/src/views/ParkingBookList.vue:487) **逻辑完全一致**：

```java
/**
 * 从车位位置文本中提取楼层 ID
 * "4号楼B1层B124" → "4F-B1"
 * "5号楼B2层B区B045" → "5F-B2"
 */
private String parseFloorId(String positionText) {
    for (java.util.Map.Entry<String, String> entry : FLOOR_MAP.entrySet()) {
        if (positionText.contains(entry.getKey())) {
            return entry.getValue();
        }
    }
    return null;
}

/**
 * 从车位位置文本中提取车位编号
 * "4号楼B1层B124"  → "B124"
 * "5号楼B2层C056"  → "C056"
 * "5号楼B2层B区B045" → "B045"（去掉 "B区" 标记）
 *
 * @param floorKey 匹配到的楼层中文键，如 "5号楼B2层"
 */
private String parseSpaceLabel(String positionText, String floorKey) {
    int idx = positionText.indexOf(floorKey);
    String raw = positionText.substring(idx + floorKey.length()).trim();
    // 5号楼B2层特殊处理：去掉 A区/B区 标记
    if ("5F-B2".equals(FLOOR_MAP.get(floorKey))) {
        raw = raw.replaceAll("^[AB]区", "").trim();
    }
    return raw;
}
```

### 4.2 新增截图步骤（在步骤 C 连接 Chrome 之后、步骤 1 登录之前）

在 `Browser browser = playwright.chromium().connectOverCDP(...)` 之后、`page.navigate(LOGIN_URL)` 之前插入：

```java
// ==================== 步骤 C2：截图车位图 ====================
String screenshotPath = null;
try {
    screenshotPath = parkingScreenshotService.takeScreenshot(browser, parkingPosition);
    if (screenshotPath != null) {
        boolean copied = parkingScreenshotService.copyToClipboard(screenshotPath);
        if (!copied) {
            log.warn("邮件通知：车位图写入剪贴板失败，将仅发文字邮件");
            screenshotPath = null; // 标记为不可用
        }
    } else {
        log.warn("邮件通知：车位图截图失败，将仅发文字邮件");
    }
} catch (Exception e) {
    log.warn("邮件通知：截图服务异常，将仅发文字邮件: {}", e.getMessage());
    screenshotPath = null;
}
```

**为什么放在登录之前？**
- 截图页（localhost:8082）无需认证，不会影响邮箱 Cookie
- 截图用一个独立的 `BrowserContext`，用完即关
- 剪贴板是系统全局的，写入后不会因为后续登录/点击操作而丢失
- 先准备好了剪贴板，后面一气呵成粘贴发送

### 4.3 改造步骤 7：正文填写

```java
/*
 * ==== 步骤7：填写正文 ====
 *
 * 优先级：
 *   1. 有截图 → 先 Ctrl+V 粘贴图片，再输入文字说明
 *   2. 无截图 → 仅输入文字（降级）
 */
log.info("邮件通知：输入正文...");

if (screenshotPath != null) {
    // ---- 方案 A：有车位截图，粘贴图片 + 文字 ----
    // 步骤 7a：聚焦正文编辑区
    // 注意：如果 #txtContent 是 <textarea>，则需要定位到其所在的
    // contenteditable 区域或嵌套 iframe。当前按照 iframe 内 #txtContent
    // 可能是 contenteditable div 的情况处理（用户确认正文是 iframe 结构）。
    composeFrame.locator(BODY_ID).click();
    page.waitForTimeout(300);

    // 步骤 7b：Ctrl+V 粘贴图片
    page.keyboard().press("Control+V");
    page.waitForTimeout(2000); // 等编辑器上传/渲染图片

    // 步骤 7c：换行后输入文字
    page.keyboard().press("Enter");
    composeFrame.locator(BODY_ID).fill(content);
} else {
    // ---- 方案 B：无截图，仅文字 ----
    composeFrame.locator(BODY_ID).fill(content);
}
```

**⚠️ 粘贴逻辑的实际 DOM 适配说明（需用户实地确认后调整）：**

1. 如果 `#txtContent` 是 `contenteditable` 的 div → 直接 `click()` + `Ctrl+V` 即可
2. 如果 `#txtContent` 是 `<textarea>` 且图片不支持 → 需要在 `#ifrm_compose_0` 内部找富文本编辑器的嵌套 iframe，类似 163 邮箱的 `iframe.APP-editor-iframe`
3. 如果正文编辑区有"插入图片"按钮 → 更复杂，暂不考虑

**当前按方案1实现**：`composeFrame.locator(BODY_ID)` 定位到 contenteditable 区域，`Ctrl+V` 粘贴。如果实测不生效，再调整为嵌套 iframe 方式。

### 4.4 异常处理

整个 try 块的结构：

```java
try {
    // ... 参数校验 ...

    // ... 步骤 A/B/C: 启动 Chrome 连接 ...

    try {
        // ==== C2: 截图（try-catch 包裹，失败不抛异常） ====
        // ... (见 4.2) ...

        // ==== 步骤 1-8: 邮箱操作 ====
        // ... 登录、写信、填收件人、填主题 ...
        // ... 步骤 7: Ctrl+V 或 fill(content) ...

    } catch (Exception e) {
        // 发送失败 → 保存错误截图
        log.error("邮件通知：发送失败 用户={}", config.getEmpNo(), e);
        try {
            page.screenshot(new Page.ScreenshotOptions()
                .setPath(Paths.get("email_error_" + config.getEmpNo() + ".png")));
        } catch (Exception ignored) {}
    } finally {
        // ---- 清理：删除截图文件 ----
        if (screenshotPath != null) {
            try {
                new File(screenshotPath).delete();
                log.info("邮件通知：截图已清理 {}", screenshotPath);
            } catch (Exception ignored) {}
        }
        context.close();
        browser.close();
    }

} catch (...) { ... }
```

### 4.5 截图文件清理

- **成功**：发送后 finally 中删除
- **失败**（catch 里抛异常）→ 也会走到 finally → 删除
- **截图服务自身失败**（`takeScreenshot` 返回 null）→ `screenshotPath == null` → 不删除（也没文件可删）

## 七、风险点与注意事项

| 风险 | 概率 | 缓解措施 |
|------|:----:|---------|
| 正文编辑器不支持图片粘贴 | 低 | 确认是 iframe + contenteditable 结构 |
| AWT Toolkit headless 不可用 | 极低 | 你是在桌面 Windows 环境运行 |
| 前端 dev server 未启动时截图失败 | 中 | catch 住 → 仅发文字邮件 |
| `Ctrl+V` 粘贴超时（图片大） | 低 | sleep(2000)，不够再加 |
| 剪贴板被其他程序覆盖 | 极低 | 截图和粘贴之间不超过 20 秒 |
| 多人并发写剪贴板 | 零 | `mailTaskExecutor` core=1 串行 |
